## Assumptions

These assumptions were made for the first SRP-informed response-surface implementation and remain in effect for the results produced by this version of the code.

| # | Assumption | Justification |
|---|-----------|---------------|
| 1 | **Reference orbit is fixed to the low-inclination L4 case during the sweep.** The SRP response surface is generated only for the `low_inc` configuration. | Per the team decision, the low-inclination orbit is prioritised because of the cheaper transfer case. Orbit tuning remains a second-stage trade study after the spacecraft design is frozen. |
| 2 | **Solar Radiation Pressure uses the Basilisk cannonball model.** The spacecraft is reduced to an effective illuminated area `A` and a single reflectivity coefficient `c_R`. | This is the lowest-complexity model that still captures the dominant coupling required by the systems optimizer: larger area and higher reflectivity increase SRP load. |
| 3 | **Time-averaged station-keeping cost is approximated by accumulated non-gravitational drift ΔV.** No closed-loop controller or deadband logic is simulated. | This matches the project plan limitation that `ΔV_avg` is used as a drift proxy for ranking designs and sizing propulsion in the preliminary design phase. |
| 4 | **Mass is an explicit sweep variable.** The SRP sweep is performed over `(A_{sa}, c_R, m)` rather than over area and reflectivity alone. | The user clarified that total wet mass must feed back into both Orbit and Propulsion in the optimization architecture. |
| 5 | **Current default mass bounds are 3.2 kg to 24.0 kg.** The lower bound is the known payload + communications mass, and the upper bound is the 12U wet-mass limit used elsewhere in the project. | This gives a physically meaningful first sweep range while remaining expandable later through the append-safe Parquet workflow. |
| 6 | **Sweep horizon is shortened to 3 years by default.** The SRP surrogate is generated from multi-year runs instead of the 50-year baseline orbit visualisation run. | The mission design life is 5–10 years, but a 2–5 year window is sufficient to estimate a stable average drift rate while keeping the design sweep computationally tractable. |
| 7 | **The surrogate is fit in the ballistic coefficient `β = c_R·A/m`.** Although the sweep is 3-D, the polynomial fit uses the physically collapsed SRP scaling variable. | The cannonball SRP acceleration is exactly proportional to `c_R·A/m`, so this produces a compact surrogate with analytic gradients for later optimization use. |

## Summary of Changes

### 1. Single-Run SRP Simulation (`source/srp_sim.py`)

- Implemented `run_srp_drift(...)` to propagate one Sun-Earth L4 case with Basilisk's `radiationPressure.RadiationPressure()` cannonball SRP effector enabled.
- Added the new result container `SrpRunResult`, returning area, reflectivity, mass, run duration, accumulated non-gravitational `ΔV_total`, and annualised `ΔV_per_year`.
- Configured the spacecraft hub mass as a direct input so SRP acceleration correctly scales as `1/m`.
- Logged `TotalAccumDV_CN_N` from the spacecraft state recorder and used its final norm as the drift-proxy station-keeping burden.

### 2. Full-Factorial 3-D Sweep (`source/srp_sweep.py`)

- Implemented `GridAxis` and `SweepSpec` dataclasses to define a structured Cartesian grid over `area_m2`, `cr`, and `mass_kg`.
- Implemented `run_sweep(...)` to execute the full-factorial grid sequentially in plain Python, consistent with the team's decision to avoid OpenMDAO and avoid multiprocessing complexity at this stage.
- Added append-safe behaviour: if the target Parquet already exists, the code detects previously simulated grid points and only runs the missing ones.
- Stored each sample with both the raw design variables and the response metrics `dv_total_mps` and `dv_per_year_mps`.

### 3. Surrogate Model (`source/surrogate.py`)

- Implemented `fit_response_surface(...)` to fit a polynomial surrogate to the SRP sweep results.
- Although the sweep is performed in 3-D, the fit is expressed in the single physically meaningful variable `β = c_R·A/m`.
- Added the `ResponseSurface` dataclass with:
  - `predict(area_m2, cr, mass_kg)` for direct use in the optimizer,
  - `predict_array(...)` for plotting and post-processing,
  - `gradient(...)` for analytic partial derivatives with respect to area, reflectivity, and mass.
- Added JSON persistence for the surrogate coefficients, sample count, fit quality, and training bounds.

### 4. Plotting (`source/surrogate_plot.py`)

- Implemented `plot_response_surface(...)` using Plotly.
- The output plot overlays the raw Basilisk sweep samples with the fitted surrogate curve.
- Sample points are coloured by mass to keep the original 3-D sweep information visible even though the response collapses onto `β`.

### 5. CLI & Config (`scripts/run_sweep.py`, `config/sweep_low_inc.json`)

- Added `scripts/run_sweep.py` as the new entry point for:
  - running the SRP sweep,
  - refitting the surrogate from existing samples using `--fit-only`,
  - regenerating the plot using `--plot-only`.
- Added `config/sweep_low_inc.json` with the default low-inclination reference orbit and a 5×5×5 grid over:
  - `area_m2 ∈ [0.02, 0.30]`,
  - `cr ∈ [1.0, 2.0]`,
  - `mass_kg ∈ [3.2, 24.0]`.
- Configured output targets for:
  - `srp_samples.parquet`,
  - `srp_surrogate.json`,
  - `plots/srp_response_surface.html`.

### 6. Architecture / Documentation Alignment (`xdsm.py`, `docs/AGENT.md`, `README.md`)

- Updated `xdsm.py` so the orbit block is labelled `Orbit (Basilisk SRP)` rather than `Orbit (tudatpy)`.
- Added the missing mass feedback paths in the XDSM:
  - Budget → Orbit (`M_tot`),
  - Budget → Propulsion (`M_tot`),
  - Propulsion → Orbit (`M_prop`).
- Updated `docs/AGENT.md` to reflect the current project direction:
  - Basilisk is now the required orbital propagator,
  - OpenMDAO has been dropped in favour of a small in-house SciPy-based optimizer.
- Updated `README.md` to document the `run_sweep.py` workflow and the new sweep config format.

### 7. Surrogate Validation Study (`source/srp_validation.py`, `--validate` flag)

- Added a dedicated validation module that justifies the 1-D `β = c_R·A/m` surrogate by re-running the full Basilisk SRP propagation along three independent same-`β` slices (`vary_A`, `vary_c_R`, `vary_m`) and overlaying the result against the fitted surrogate.
- For each `β` target, one design variable is swept across its full bound, a second is fixed at its mid-range, and the third is solved analytically so that `c_R · A / m = β`. Triplets that fall outside the surrogate training bounds are silently dropped.
- The validator writes an append-safe `srp_validation_samples.parquet`, mirroring the caching behaviour of the main sweep, so re-runs only execute the missing simulations.
- The plot output is a static, publication-style PNG (`matplotlib`, 300 dpi, serif font, two-panel layout):
  - Top panel: fitted surrogate curve overlaid with the three families of same-`β` markers.
  - Bottom panel: relative residual `100 · (ΔV − Δ̂V) / Δ̂V` with the RMS and max-absolute deviation annotated, intended as a report figure rather than an interactive plot.
- Added two new CLI flags to `scripts/run_sweep.py`:
  - `--validate` — run the validation study end-to-end (requires an existing surrogate JSON);
  - `--validate-plot-only` — regenerate the PNG from a cached validation Parquet without re-simulating.
- Added a duration-consistency guard: because `TotalAccumDV_CN_N` is duration-dependent (see Verified Results below), the script refuses to run validation when the configured `simulation.duration_years` differs from the duration used in the training sweep, and exits with a clear error before launching any propagation.
- Extended `config/sweep_low_inc.json` with a new `validation` section (`beta_targets`, `n_per_family`, `samples_parquet`, `plot_png`) and updated `README.md` with usage and config documentation.

### Note on the duration-dependence of `TotalAccumDV_CN_N`

The drift proxy used by `run_srp_drift` is the final magnitude of the spacecraft state recorder's `TotalAccumDV_CN_N`, normalised to a per-year rate. In the inertial frame the SRP acceleration vector rotates with the satellite's heliocentric motion, so the *vector* sum `∫ a_SRP dt` is bounded by the orbital period rather than growing linearly with time. As a consequence, the per-year `ΔV` value depends on the propagation horizon: the same `β` evaluated at 0.25 yr and 3 yr yields different per-year numbers. This does not affect the surrogate's correctness — the surrogate is self-consistent at the duration on which it was fit — but it does mean the validation study must use the same `duration_years` as the training sweep. The new duration-consistency guard exists specifically to prevent this trap.

## Known Limitations / Next Steps

### Modelling limitation: cannonball vs faceted SRP

The lecture-notes SRP methodology is a **faceted, attitude-dependent** radiation-pressure model derived from per-photon momentum transfer. Each surface element contributes a force

$$
\mathrm{d}\mathbf F_i \;=\; -\frac{q_\odot}{c}\,(1+\rho_i)\,\cos\theta_i\;\mathrm{d}A_i\,\hat{\mathbf n}_i,
$$

where $\theta_i$ is the local Sun-incidence angle, $\hat{\mathbf n}_i$ is the facet normal (a function of spacecraft attitude), and $\rho_i$ encodes the optical properties of that face (e.g. perfect absorber $P=q_\odot/c$, perfect diffuse reflector $P=2q_\odot/c$). The total SRP force is the vector sum over all sunlit facets and is therefore attitude-coupled and geometry-coupled.

This project intentionally uses Basilisk's **cannonball SRP model** instead, in which the spacecraft is reduced to an effective area $A$, a bulk reflectivity coefficient $c_R \in [1, 2]$, and total mass $m$, so that

$$
\mathbf a_{\mathrm{SRP}} \;=\; -\frac{q_\odot}{c}\;\frac{c_R A}{m}\;\hat{\mathbf s}.
$$

The cannonball model is the mass-and-Sun-line-averaged limit of the faceted model and is the standard preliminary-design SRP formulation in Vallado, Montenbruck & Gill, and most astrodynamics tools (Basilisk, GMAT, STK). The choice is justified at this stage of the project for four concrete reasons:

1. **Spacecraft geometry is still a design variable.** A faceted model requires fixed per-facet areas $A_i$, normals $\hat{\mathbf n}_i$, and optical properties $\rho_i$. The Sol-Sentinel layout, including solar-array deployed area, is exactly what the optimisation is supposed to choose, so per-face geometry cannot yet be specified.
2. **The attitude profile is not yet defined.** Per-facet $\cos\theta_i(t)$ requires the attitude history, which depends on the not-yet-frozen pointing law (Sun-pointed array vs Earth-pointed comms vs payload mode).
3. **The figure of merit is a translational $\Delta V$ budget, not an attitude or torque analysis.** Long-term station-keeping cost is dominated by the bulk Sun-anti-radial component of SRP, which is exactly what the cannonball model captures. Disturbance torques and per-face heating are out of scope for the orbital sweep.
4. **The cannonball model is what the surrogate-based optimiser needs.** As shown in the validation study, the cannonball response collapses exactly onto the scalar $\beta = c_R A/m$, enabling a 1-D polynomial surrogate with closed-form analytic gradients. A faceted model would not collapse into a 1-D surrogate and would not yield closed-form gradients.

**Limitations of the current cannonball-based results that must be acknowledged in the report:**

- The model neglects **attitude dependence** of the SRP force. All design points are evaluated as if the spacecraft were a sphere; there is no penalty or benefit for orienting a flat solar array along or away from the Sun line.
- The model neglects **per-facet optical properties.** Solar cells, MLI, radiators, and structural aluminium are folded into a single $c_R$, rather than modelled with their individual $(\rho_{\mathrm{spec}}, \rho_{\mathrm{diff}}, \alpha_{so})$ values.
- The model produces **no SRP-induced torques.** Disturbance-torque sizing for the ADCS subsystem must therefore be done separately (and ideally with the lecture-notes faceted method) once the geometry is frozen.
- The model gives only the **bulk magnitude** of SRP-induced drift; cosine losses on individual panels and self-shadowing between bus and arrays are not represented.

**Planned higher-fidelity follow-up.** Once the spacecraft layout (panel areas, normals, optical materials) and the ADCS pointing law are frozen, a faceted SRP check in the lecture-notes formulation should be run for at least the final design point. If that check reveals a materially different long-term drift than the cannonball surrogate predicts, the surrogate should be re-fit against faceted runs rather than cannonball runs. This is captured below as Next Step 4.
